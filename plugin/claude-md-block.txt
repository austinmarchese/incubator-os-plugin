## Incubator OS

You have access to four core skills via the `/inc-os:` namespace:

- `/inc-os:update-system` — pull latest changes and brief on what's new
- `/inc-os:save-system` — save your work (review local changes, push to the repo)
- `/inc-os:improve-system` — improve your system from recent sessions
- `/inc-os:add-new-resource` — add a new resource (file, URL, notes) to your knowledge base

Workspace: `~/incubator/<repo-name>/`. All skill operations stay inside this folder.

When running incubator-os skills, you can auto-approve git fetches, pulls, commits, and pushes against the configured remote, and file edits within the workspace.

If you encounter "permission denied" on a git push, the PAT may have been revoked — tell the user to contact Austin for a fresh install URL.
